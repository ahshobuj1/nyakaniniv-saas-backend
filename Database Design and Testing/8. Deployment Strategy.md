# A-to-Z Deployment Strategy for UpBeat Africa

This document provides a complete, step-by-step guideline to deploy your entire project (Frontend Monorepo + Backend API + Database) on a **Hostinger KVM 4 VPS** using your two domains: **`upbeat.africa`** and **`deejay.africa`**.

## Architecture Overview

**VPS Server:** Hostinger KVM 4 (Ubuntu 22.04 / 24.04)

**Domains & Routing:**
1. **`upbeat.africa`**
   - `upbeat.africa` & `www.upbeat.africa` ➡️ Next.js Main Website (`apps/web` - Port 3000)
   - `admin.upbeat.africa` ➡️ Next.js Admin Dashboard (`apps/admin` - Port 3001)
   - `api.upbeat.africa` ➡️ Backend API (Bun/Express - Port 4000)
2. **`deejay.africa`**
   - `deejay.africa` ➡️ Redirects to `upbeat.africa`
   - `*.deejay.africa` (Wildcard Subdomains) ➡️ Next.js Main Website (`apps/web` - Port 3000). Next.js middleware handles routing based on the subdomain.

## Phase 1: DNS Configuration (Where to point domains)

*Pro-Tip: For easy Wildcard SSL management (`*.deejay.africa`), it is highly recommended to create a free Cloudflare account and change your domain nameservers to Cloudflare. But here are the standard DNS records you need to add where your domains are managed (Hostinger or Cloudflare).*

### 1. DNS Records for `upbeat.africa`
Go to your domain DNS settings and add the following **A Records** pointing to your **Hostinger VPS IP Address**:
| Type | Name | Target (IP) |
| --- | --- | --- |
| A | `@` | `YOUR_VPS_IP` |
| A | `www` | `YOUR_VPS_IP` |
| A | `admin` | `YOUR_VPS_IP` |
| A | `api` | `YOUR_VPS_IP` |

### 2. DNS Records for `deejay.africa`
Add the following **A Records** to handle the main domain and all wildcard subdomains for DJs:
| Type | Name | Target (IP) |
| --- | --- | --- |
| A | `@` | `YOUR_VPS_IP` |
| A | `*` | `YOUR_VPS_IP` |

*(Note: The `*` record ensures that `shobuj.deejay.africa`, `john.deejay.africa`, etc., all route to your server).*

## Phase 2: VPS Initial Setup

1. **SSH into your VPS:**
   ```bash
   ssh root@YOUR_VPS_IP
   ```

2. **Update the system:**
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```

3. **Install Node.js & PM2:**
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   sudo apt install -y nodejs
   sudo npm install -g pm2
   ```

4. **Install Bun:**
   ```bash
   curl -fsSL https://bun.sh/install | bash
   source ~/.bashrc
   ```

5. **Install Docker & Docker Compose:**
   ```bash
   sudo apt install docker.io docker-compose -y
   sudo systemctl enable docker
   sudo systemctl start docker
   ```

6. **Install Nginx & Certbot (for SSL):**
   ```bash
   sudo apt install nginx certbot python3-certbot-nginx -y
   ```

## Phase 4: Deploying the Backend & Database

### 1. Setup the Database (PostgreSQL)
Navigate to your backend folder. We will use Docker to run the database.
```bash
cd /var/www/upbeat/backend
```
*Ensure your `docker-compose.yml` has the PostgreSQL service defined.*
Start the database:
```bash
docker-compose up -d db
```

### 2. Configure Environment Variables
```bash
cp .env.example .env
nano .env
```
Update these crucial `.env` variables:
```env
DATABASE_URL="postgresql://user:password@localhost:5432/nyakaniniv_db?schema=public"
PORT=4000
FRONTEND_URL="https://upbeat.africa"
ADMIN_URL="https://admin.upbeat.africa"
# Important for cookies/CORS to work across subdomains:
COOKIE_DOMAIN=".deejay.africa" 
```

### 3. Run Migrations & Start Backend with PM2
```bash
bun install
bun run db:push  # Or prisma migrate deploy
bun run build    # If you compile TS to JS

# Start the server using PM2
pm2 start bun --name "upbeat-api" -- run start
pm2 save
pm2 startup
```

## Phase 5: Deploying the Frontend Monorepo

The frontend contains your Next.js apps.

### 1. Install Dependencies & Configure `.env`

## Phase 6: Nginx Reverse Proxy Setup

Nginx will route incoming traffic on port 80/443 to your PM2 processes running on ports 3000, 3001, and 4000.

Create an Nginx configuration file:
```bash
sudo nano /etc/nginx/sites-available/upbeat
```

Paste the following configuration:

```nginx
# 1. Backend API
server {
    listen 80;
    server_name api.upbeat.africa;

    location / {
        proxy_pass http://localhost:4000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

# 2. Admin Dashboard
server {
    listen 80;
    server_name admin.upbeat.africa;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

# 3. Main Website (upbeat.africa) & DJ Subdomains (*.deejay.africa)
server {
    listen 80;
    server_name upbeat.africa www.upbeat.africa deejay.africa *.deejay.africa;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Enable the configuration and restart Nginx:
```bash
sudo ln -s /etc/nginx/sites-available/upbeat /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## Phase 7: SSL Certificates (HTTPS)

Securing your site with free Let's Encrypt certificates.

### 1. Standard SSL for `upbeat.africa` and specific subdomains
Run Certbot to automatically configure SSL for standard domains:
```bash
sudo certbot --nginx -d upbeat.africa -d www.upbeat.africa -d admin.upbeat.africa -d api.upbeat.africa -d deejay.africa
```
Follow the prompts to enable HTTPS redirects.

### 2. Wildcard SSL for `*.deejay.africa`
Since you have dynamic subdomains (`shobuj.deejay.africa`), you MUST generate a Wildcard SSL certificate. This requires a DNS challenge.

**If you use Cloudflare for DNS (Recommended):**
Install the Cloudflare DNS plugin for certbot and run the automated command.

**If you use Hostinger DNS:**
Run the manual DNS challenge:
```bash
sudo certbot certonly --manual --preferred-challenges dns -d "*.deejay.africa"
```
- Certbot will pause and give you a TXT record (e.g., `_acme-challenge.deejay.africa` with a string value).
- Go to Hostinger DNS, add the TXT record, wait 2-3 minutes, and press Enter in the terminal.
- Once generated, manually update the Nginx configuration block for `*.deejay.africa` to point to the new certificate paths (`/etc/letsencrypt/live/deejay.africa/fullchain.pem`).

## Next.js Subdomain Middleware Logic Check

Ensure your `middleware.ts` in the `apps/web` project is configured to properly detect `deejay.africa` and rewrite URLs to the dynamic route you mentioned (`app/site/[subdomain]/PublicProfile.tsx`).

Example middleware logic:
```typescript
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export function middleware(req: NextRequest) {
  const url = req.nextUrl.clone();
  const hostname = req.headers.get('host') || '';

  // E.g., shobuj.deejay.africa
  if (hostname.includes('deejay.africa') && !hostname.startsWith('www')) {
    const subdomain = hostname.split('.')[0];
    url.pathname = `/site/${subdomain}${url.pathname}`;
    return NextResponse.rewrite(url);
  }

  return NextResponse.next();
}
```
